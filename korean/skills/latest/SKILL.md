# /latest — 세션 종료 메모리 업데이트

현재 세션 작업을 기반으로 모든 컨텍스트 파일을 업데이트한다.
이 스킬이 제대로 안 되면 다음 세션에서 맥락이 끊긴다. 가장 중요한 스킬이다.

---

## Phase 1: 세션 분석

현재 대화를 쭉 훑어서 아래를 정리한다. 빠뜨리면 다음 세션에서 모른다.

- **완료한 작업** — 이번 세션에서 뭘 했는지. 구체적으로.
- **발견한 사실** — 새로 알게 된 것. 확정된 결론.
- **실험 결과** — 돌린 게 있으면 결과와 해석.
- **남은 작업** — 다음에 이어서 할 것. 구체적 다음 스텝.
- **블로커** — 막힌 게 있으면 뭐가 막혔는지.
- **결정 사항** — 방향 전환, 설계 변경 등.

---

## Phase 2: 파일 업데이트

**절대 규칙: 덮어쓰기 전에 이전 버전을 `context/archive/`에 먼저 옮겨라.**
파일명 형식: `{원래이름}_{YYYY-MM-DD}_{HHMM}.md` (한국 시간 기준, 24시간)
예: `short_term_2026-03-28_1430.md`

### 2a. `context/short_term.md` — 항상 갱신

이전 버전 archive → 새로 작성. 양식은 `short-term-template.md` 참고.

**검증:** 다 쓴 후 다시 읽어보고, 이것만 읽고 작업을 이어갈 수 있는지 자문해라. 모호하면 다시 써라.

### 2b. `context/long_term.md` — 확정 결론 있을 때만

- 이번 세션에서 확정된 결론이 있으면 → 증거와 함께 추가.
- 방향이 바뀌었으면 → 이전 파일 archive 후 새로 작성. 왜 바꿨는지 반드시 기록.
- 변한 게 없으면 → 건드리지 마라.

### 2c. `MEMORY.md` — 새 영구 지식 있을 때만

- 새로운 API 발견, 확인된 동작, 실패 기록 등 → 추가.
- 이미 있는 내용과 중복 확인 먼저.
- 휘발성 상태 넣지 마라. 안정적 사실만.
- 변한 게 없으면 → 건드리지 마라.

### 2d. `CLAUDE.md` — 상태 바뀌었을 때만

- 현재 상태/단계, 기술 스택, 제약사항이 바뀌었으면 → 반영.
- 변한 게 없으면 → 건드리지 마라.

### 2e. `context/exp_log.md` — 실험 했을 때만

실험 돌렸으면 기록. 양식은 `exp-log-template.md` 참고. 실험 안 했으면 건드리지 마라.

## Phase 3: GPT Pro 프롬프트

**inbox 절대 보지 마라.** inbox는 `/fresh` 전용이다.

1. `context/gpt-pro/outbox/` glob (파일명만, 내용 읽지 마라)
2. **대기 중인 요청 있으면** → "Pro 대기 중: {주제}" 보고하고 건너뛴다.
3. **비어있으면** → 이번 세션에서 벽에 부딪힌 질문이 있는지 확인.
   - 있으면 → `pro_request_{YYYY-MM-DD}_{topic}.md` 작성. 양식은 `gpt-pro-prompt-template.md`, 규칙은 `gpt-pro-rules.md` 참고.
   - 없으면 → "Pro 건너뜀 — 보낼 만한 질문 없음" 보고.

---

## Phase 4: 로그 동기화

서버에서 실험 돌렸으면:

```bash
bash scripts/sync_logs.sh   # 서버 logs/ → 로컬 logs/
```

안 했으면 건너뛴다. 했는데 sync 스크립트 없으면 → 사용자에게 수동 동기화 알림.

---

## Phase 5: Stale 체크

각 파일의 "현재 상태" 섹션이 실제 현실과 맞는지 검증:

- CLAUDE.md 현재 상태가 정확한가?
- short_term.md 다음 스텝이 아직 유효한가?
- long_term.md 방향이 아직 활성인가?
- MEMORY.md 최고 결과가 최신인가?

**안 맞으면 즉시 수정해라.** stale한 컨텍스트는 다음 세션을 오염시킨다.

---

## Phase 6: 보고

```
## 세션 종료 보고

[short-term]  갱신됨 / 변경없음
[long-term]   추가: {뭘 추가했는지} / 변경없음
[permanent]   추가: {뭘 추가했는지} / 변경없음
[CLAUDE.md]   갱신됨 / 변경없음
[exp-log]     추가: {실험명} / 변경없음
[gpt-pro]     프롬프트 작성: {주제} / 대기 중: {주제} / 건너뜀
[logs]        동기화 완료 / 해당없음
[stale]       수정함: {뭘} / 전부 최신

요약: {이번 세션에서 한 것 한 줄}
다음: {다음 세션에서 할 것 한 줄}
```

---

## 상태 보고

- **DONE** — 모든 파일 업데이트 완료. 다음 세션 연속성 확보됨.
- **DONE_WITH_CONCERNS** — 업데이트했지만 문제 있음. 구체적으로 명시.
- **BLOCKED** — 업데이트 불가. 사유 명시.

---

## 주의

- 건드릴 필요 없는 파일은 건드리지 마라. "변경없음"이 정상이다.
- 덮어쓰기 전 archive는 절대 규칙이다. 빼먹으면 이전 맥락이 영구 소실된다.
- 이 스킬은 세션 끝에 한 번만 실행한다. 중간에 돌리지 마라.
