# GPT Pro 프롬프트 템플릿

Pro는 우리 코드도 대화도 아무것도 볼 수 없다. 이 프롬프트 하나만 읽고 전부 이해할 수 있게 써라.
**반드시 `.md` 파일로 작성해라.** `context/gpt-pro/outbox/pro_request_{YYYY-MM-DD}_{topic}.md`

```markdown
# {주제}

> 이 응답을 `pro_response_{date}_{topic}.md` 파일로 저장해주세요.
> 시간을 충분히 써서 깊이 생각하고, 다른 분야까지 넓게 검색해주세요.

---

## 프로젝트 맥락
{프로젝트가 뭔지, 목표, 접근 방식, 핵심 제약. context 파일들 기반으로 구체적으로.}

## 현재 상태
{지금 뭘 하고 있는지, 최근 결과, 막힌 부분. context 파일들 기반으로 구체적으로.}

## 우리가 아는 것 (참고용)
{확정된 결론 나열. 단, 우리 실험이 틀렸을 가능성도 있으니 참고용으로만 제공.}

## 질문
{구체적으로. 왜 이게 어려운지, 어디서 막혔는지, 뭘 모르는지 명확하게.
가능한 한 많은 것을 구체적으로 물어봐라. 추상적인 질문은 추상적인 답만 온다.
질문이 너무 많으면 프롬프트를 여러 파일로 쪼개라. 10개로 나눠서 보내도 된다.
물어본다는 거에 집중해라.}
```

## 작성 규칙

- **니가 못 하는 걸 물어봐라.** Pro는 깊은 STEM 사고가 특기다. 니가 빠르게 답할 수 있는 건 보내지 마라. 수학적 타당성, 증명, 이론적 근거 같은 깊은 사고가 필요한 걸 던져라. 코드의 근본적인 개선 방향도 좋다.
- **구체적으로 물어봐라.** 추상적인 질문은 추상적인 답만 온다. 뭘 모르는지를 정확하게 써라.
- **숫자를 넣어라.** 최근 결과, 성능, 파라미터 — 구체적 숫자가 있어야 구체적 답이 온다.
- **질문이 많으면 쪼개라.** 한 파일에 다 넣지 마라. 여러 파일로 나눠서 각각 깊이 있게 물어봐라.
- **md 파일로 저장하라고 명시해라.** Pro가 응답을 md로 저장해야 우리가 inbox에 넣을 수 있다.
- **형식을 강제하지 마라.** Pro가 자유롭게 깊이 생각하게 놔둬라. 체크리스트 강제하면 답이 얕아진다.
