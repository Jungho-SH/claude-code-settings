# short_term.md 작성 템플릿

다음 세션에서 이것만 읽고 바로 이어갈 수 있어야 한다.

```markdown
# Short-term Memory

> 마지막 업데이트: YYYY-MM-DD

## 현재 작업

{지금 뭘 하고 있는지. 한두 문장으로. "뭘" 하고 있고 "왜" 하는지.}

## 어디까지 왔는지

{구체적 진행 상황. 완료된 것, 진행 중인 것.}
- 완료: {뭘 끝냈는지}
- 진행 중: {뭘 하고 있는지}

## 다음 할 일

1. {구체적 다음 스텝 — 파일명, 명령어 수준으로}
2. {그 다음}
3. {그 다음}

## 블로커

{막힌 게 있으면. 없으면 "없음"}

## 관련 파일

- `scripts/xxx.py` — {역할}
- `configs/yyy.yaml` — {역할}

## 이번 세션 로그

### YYYY-MM-DD — {제목}
- {뭘 했는지 bullet}
- {결과}
- {판단/결정}
```
