# /fresh 보고 템플릿

보고를 읽은 사용자가 아무것도 추가로 읽지 않아도 바로 "그래 이거 하자"라고 말할 수 있어야 한다.
빈 말 금지. 구체적으로 써라.

```markdown
## 지난 세션 요약

{지난 세션에서 뭘 했는지. 핵심만 2-3줄.}

## 지금 해야 할 일

1. {가장 먼저 해야 할 것}
2. {그 다음}
3. {그 다음}

## 주의할 점

{블로커, stale 경고, 충돌 등 있으면 여기. 없으면 섹션 자체를 생략.}

## Pro 인사이트

{GPT Pro 응답이 있었으면 핵심 인사이트 정리. 없었으면 섹션 자체를 생략.}
```

없는 섹션은 빈 칸으로 남기지 말고 아예 빼라. 쓸데없는 정보로 사용자 시간 낭비하지 마라.
