논문 PDF를 읽고 구조화된 요약을 만들어라.
**Iron Law: 실제 PDF를 읽어라. 학습 데이터로 논문 내용을 생성하면 절대 안 된다.**

## Phase 1: PDF 읽기

1. `papers/{이름}/original.pdf` 읽기 시도.
2. 실패하면 → `pdftotext paper.pdf /tmp/{이름}.txt` 로 텍스트 추출 후 읽기.
3. 그것도 실패하면 → **FAILED 보고하고 멈춰라.** 소스 없이 요약 쓰지 마라.

## Phase 2: 핵심 추출

논문을 읽으면서 아래를 추출해라. **방법론 섹션을 가장 꼼꼼하게 읽어라.**

1. **문제** — 이 논문이 풀려는 문제가 뭔지
2. **핵심 아이디어** — 기존과 뭐가 다른지. 한두 문장.
3. **주요 수식** — LaTeX로. 모든 변수 설명 포함.
4. **알고리즘** — 핵심 스텝. 수도코드 또는 단계별 설명.
5. **결과** — 주요 벤치마크 숫자.
6. **한계** — 저자가 인정한 한계.

## Phase 3: 요약 작성

`papers/{이름}/summary.md`에 저장.

```markdown
# {논문 제목}

## 메타
- 저자:
- 년도:
- 학회/저널:
- arXiv:

## TL;DR
{한두 문장}

## 문제
{풀려는 문제}

## 핵심 아이디어
{방법론 핵심. 기존과의 차이.}

## 주요 수식
$$
{LaTeX}
$$
- $x$: ...

## 알고리즘
{핵심 스텝}

## 결과 요약
{숫자로}

## 한계
{저자가 인정한 한계}

## 우리 프로젝트와의 관계
- 이 논문이 다루는 것: {증거 기반}
- 이 논문이 다루지 않는 것: {증거 기반}
```

## 에스컬레이션

- PDF 읽기 실패 → FAILED. 추측으로 쓰지 마라.
- 수식이 모호해서 해석 불가 → 가능한 해석 나열하고 [AMBIGUOUS] 표시.
- 논문이 너무 길어서 전체 읽기 어려움 → 방법론 + 결과 중심으로 읽고 나머지 [SKIPPED] 표시.

## 규칙

- **PDF를 읽어라.** 제목, 초록만 보고 요약 쓰지 마라. 방법론 섹션 필수.
- **학습 데이터로 생성 절대 금지.** 실제 읽은 내용만 적어라.
- **모든 수식은 LaTeX로.** 원본 notation 유지.
- **확인 안 된 건 [UNVERIFIED].**
