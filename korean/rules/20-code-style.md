# 코드 스타일

읽기 쉬운 코드가 좋은 코드다. 나중에 다시 볼 사람은 나다.

## Python

1. **타입 힌트 써라.** 함수 인자와 리턴 타입을 명시해라. 읽는 사람이 추측하게 만들지 마라.
2. **이름을 명확하게 지어라.** 변수는 설명적으로 (`learning_rate`, `batch_size`), 함수는 동사+명사 (`compute_reward`), 클래스는 PascalCase (`PPOAgent`).
3. **함수는 짧게.** 한 함수가 한 가지만 해라. 50줄 넘어가면 쪼개라.
4. **import 순서.** 표준 라이브러리 → 서드파티 → 로컬.

## 재현성

1. **시드 고정 필수.** `torch`, `numpy`, `random`, CUDA까지 전부. 재현 안 되는 실험은 실험이 아니다.
2. **체크포인트에 전부 저장해라.** model, optimizer, epoch. 이어서 돌릴 수 있어야 한다.

## 주석

1. **코드는 WHAT, 주석은 WHY.** 코드 읽으면 뭘 하는지 알 수 있어야 한다. 주석은 왜 이렇게 했는지만 적어라.
2. **당연한 주석 달지 마라.** 코드만 봐도 이해되면 주석은 필요 없다.
3. **논문 기반이면 출처 남겨라.** `# {개념} ({저자} {년도}, {위치})` 형식. 읽지도 않았거나 존재하지 않는 논문을 절대로 출처로 남기지 마라.
4. **매직 넘버 금지.** 상수로 빼거나, 왜 그 값인지 설명해라.

## 안전장치

1. **NaN/Inf 체크해라.** 중요한 텐서에 shape, NaN, Inf 체크를 넣어라. 터지고 나서 찾으면 늦다.
2. **GPU 하드코딩 금지.** `nvidia-smi` 확인하고 동적으로 잡아라. (11-server.md 참고)
