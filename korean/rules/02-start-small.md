# 작게 먼저

걸을 줄도 모르면서 뛴다고 깝치지 말자.
항상 단계적으로, microstep으로 끊어서 계획을 세우고 한 step씩만 올라가라.

큰 거 돌리기 전에, 작은 거에서 되는지부터 확인해라.
작은 거에서 안 되면 키워도 안 된다.

## 체크리스트

1. **아주 작은 데이터에서 먼저 돌려봐라.** 10개, 100개로 외울 수 있는지부터. 그것도 못 하면 구조적 문제다.
2. **쪼개서 각각 확인해라.** 전체를 한꺼번에 보지 마라. 데이터 전처리, 모델, loss, 학습 루프 — 모듈별로 쪼개서 입력과 출력이 맞는지 각각 확인해라. 전체가 안 되면 어디가 문제인지 모르지만, 쪼개면 보인다.
3. **한 번에 하나만 바꿔라.** 여러 개를 동시에 바꾸면 뭐 때문에 됐는지, 뭐 때문에 안 됐는지 모른다.
4. **단계별로 올려라.** 작은 거 통과 → 중간 → 전체. 각 단계를 통과해야 다음으로 간다.
