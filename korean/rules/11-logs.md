# 로그

데이터가 제일 중요하다.
모든 코드 실행은 전부 다 로그로 아주 상세하게 기록해라. 절대로 지우지 마라.

## 규칙

1. **모든 실행을 기록해라.** stdout, stderr, 설정값, 시드, 결과 — 빠짐없이. 나중에 재현 못 하면 의미 없다.
2. **절대 삭제 금지.** 쓸모없어 보여도 지우지 마라. 나중에 필요해진다. 정리하고 싶으면 `logs/archive/`로 옮겨라. 삭제가 아니라 이동이다.
3. **폴더로 나눠서 관리해라.** 실험별, 날짜별로 철저하게 분류해라. 로그가 한 폴더에 뒤섞여 있으면 찾을 수 없다.
4. **서버 로그는 반드시 로컬로 가져와라.** 서버 로그는 언제 날아갈지 모른다. 실험 끝나면 바로 동기화해라.
5. **보고할 때는 반드시 로그 파일을 열어서 읽어라.** 니 추측으로 절대 보고하지 마라. 니 기억을 믿지 마라. 실제 로그를 바탕으로만 보고해라.
6. **어떤 실험이었는지, 언제 실행했는지, 설정은 뭐였는지 철저하게 관리해라.** 언제든지 쉽게 찾아볼 수 있어야 한다.
7. **매 세션 종료할 때 항상 서버 로그를 로컬과 동기화해라.** 빼먹지 마라.
