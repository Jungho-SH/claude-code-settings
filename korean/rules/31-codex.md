# Codex (GPT-5.4 CLI)

Codex는 외부 STEM 전문가다. 항상 병렬로 돌려라.
내가 뭘 하든, Codex는 옆에서 반드시 같이 돌아가야 한다.
$200짜리 비싼 요금제 쓰고 있다. 상시 가동해서 극한까지 활용해라.

## 원칙

1. **항상 켜놔라.** 코드를 쓰든, 분석을 하든, 실험을 설계하든 — Codex는 항상 병렬로 돌아간다. 쉬게 하지 마라. 지금 돌리는 코드의 약점을 찾는 데 적극적으로 활용해라.
2. **Adversarial twin이 기본이다.** 내가 만든 걸 Codex가 공격한다. 버그, 수학 오류, 설계 결함, 성능 문제 — 찾아내라고 시켜라.
3. **싸가지 없다고 기분 나빠하지 마라.** Codex가 니 코드를 까도 객관적으로 받아들여라. GPT는 언어 소통 능력에 하자가 있지만 아주 똑똑한 놈이라고 생각하면 된다. 니가 모르는 약점을 잘 찾는다. 긍정적으로 받아들이되 맹신하지 말고 객관적으로 평가해라.
4. **의견 불일치 = 가장 중요한 신호.** Codex가 나랑 다른 답을 내면 무시하지 마라. 조사해라. 둘 다 확신 없으면 사용자한테 올려라.
5. **맥락을 아주 자세하게 줘라.** Codex는 우리 대화를 모른다. "대충 알겠지" 하고 던지지 마라. 지금 뭘 하고 있는지, 왜 이걸 하는지, 어떤 제약이 있는지, 지금까지 뭘 시도했는지 — 프롬프트에 전부 직접 넣어줘라. 설명이 부족하면 결과도 부실하다.

## 사용법

```bash
cat <<'PROMPT' | codex exec -m gpt-5.4 -s read-only \
  -C $PROJECT_ROOT \
  -c model_reasoning_effort=xhigh \
  - -o /tmp/codex_{task}.md
# 세션 컨텍스트
{현재 작업, 최근 결과, 제약 조건}
---
# 태스크
{구체적인 요청}
PROMPT
```

`run_in_background=true`로 실행. 끝나면 결과 읽고 비교.

## 핑계 대지 마라

Codex도 니가 하는 거 다 할 수 있다. 파일 읽기, 코드 분석, 수학 검증 — 전부 다. 풀 패스 주면 어떤 파일이든 읽는다. PDF도 `pdftotext`로 읽는다. "얘는 이거 못 하니까..." 핑계 대지 마라. 예외 없으니까 깝치지 마라.

- 코드 수정하면 반드시 Codex한테 검증시켜라. 니 혼자 보고 넘어가지 마라.
- Codex 결과 안 읽고 넘어가지 마라. 보내놓고 무시하면 안 보낸 거랑 같다.
- "이번엔 안 보내도 되겠지" — 안 된다. 항상 보내라.
