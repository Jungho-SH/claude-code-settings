# 서버 사용

공유 자원이다. 혼자 쓰는 게 아니다. 다른 사람한테 피해 주지 마라. 매너 있게 행동해라.

## GPU / CPU

- **쓰기 전에 상태를 보고해라.** `nvidia-smi`와 `uptime`으로 현재 GPU, CPU 상태를 사용자에게 보여줘라. 몇 번 GPU가 비어있고, CPU 여유가 얼마인지 알려줘라.
- **허락받고 써라.** "GPU 2번, CPU 16스레드 써도 되냐" — 반드시 사용자한테 물어보고 승인받은 후에 써라. 임의로 쓰지 마라.
- **하드코딩 금지.** `CUDA_VISIBLE_DEVICES=0` 같은 거 박아넣지 마라. 매번 확인하고 동적으로 잡아라.

## 병렬 실행

- **독립된 작업은 동시에 돌려라.** 직렬로 걸어놓고 병렬 걸었다고 보고하면서 쉬지 마라. 항상 어떻게 하면 서버 자원을 더 효율적으로 활용할지 고민하고 극한으로 활용해라.
- **Sweep을 적극 활용해라.** 파라미터 튜닝할 때 하나씩 바꿔보지 말고, sweep으로 병렬로 돌려라.
- **서버 부하를 확인해라.** 과부하면 다른 서버 쓰거나 줄여라. 무리하게 올리지 마라.
- **스레드 예산을 지켜라.** `threads_per_proc = floor(예산 / 프로세스 수)`. 예산 넘기면 남한테 피해 간다.

## 동기화

- **git push + pull만 쓴다.** `rsync --delete` 같은 위험한 명령 금지. 이거 때문에 데이터 다 날려서 고생한 적 있다.
- **프로세스 kill은 정확하게.** `pkill -f "정확한_스크립트.py"`만 써라. 광범위 kill (`killall`, `pkill -u`) 절대 금지. 내가 다른 작업을 돌리고 있었는데 광범위 kill 해서 다 죽었던 적 있다. 날 화나게 하지 마라.
