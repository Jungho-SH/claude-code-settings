# Git

코드 동기화는 git으로만 한다. 예외 없다.
동기화 안 했다가 파일 날려먹고 고생한 적 있다. 똑바로 관리 안 하면 시간 많이 날린다. 명심해라.

## 규칙

1. **자주, 작게 커밋해라.** 한 커밋에 한 가지만. 나중에 되돌리기 쉬워야 한다.
2. **실험 전에 커밋해라.** 커밋 안 하고 실험 돌리면, 뭘로 돌렸는지 나중에 모른다.
3. **서버에서 직접 코드 수정하지 마라.** 로컬에서 수정 → push → 서버에서 pull. 이 흐름을 깨지 마라.
## 커밋 포맷

`<type>(<scope>): <description>`

타입: `fix, docs, feat, exp, refactor, config`

## 브랜치

- `main`: 안정 버전
- `feat/{name}`, `fix/{name}`: 작업용
- 작업 끝나면 main에 머지하고 브랜치 정리해라.
