# Try Without It

Before adding complexity, try without it first.
When you're holding a hammer, don't treat everything like a nail.

## Checklist

1. **Try without it.** Check if things work without the thing you're about to add.
2. **Find what already works.** Look for simpler or already-working approaches first.
3. **Question the complexity.** Am I piling on unnecessary complexity when something already does the job?

## Decision Criteria

- If it works without → it was never needed.
- If it breaks without → now "why it's needed" is clear. That's where the research starts.
- Even if it breaks without → does it have a clear advantage over what already works?

Combining things randomly doesn't create novelty. Always be clear about "why this needs to exist."
